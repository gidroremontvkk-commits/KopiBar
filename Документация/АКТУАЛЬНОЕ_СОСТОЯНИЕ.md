# KopiBar: актуальное состояние

## Коротко

Сейчас проект работает только с Binance Futures.

Раньше интерфейс и документация описывали вариант на 5 бирж, но от него временно отказались. Причина простая: на VPS не хватает ресурсов для хранения и обслуживания полной истории по всем монетам сразу.

Текущие ограничения VPS:

- `1 CPU`
- `1 GB RAM`
- `10 GB` диска

## Текущая схема

```text
Браузер (React) -> сервер KopiBar на VPS -> Binance Futures API
```

## Где что лежит

- фронтенд: `D:\KopiBar`
- локальная копия сервера: `D:\kopibar-server`
- боевой сервер на VPS: `/root/kopibar-server`

Локальная папка `D:\kopibar-server` используется как рабочая копия. После изменений файлы вручную копируются на VPS.

## Что делает фронтенд

- показывает список монет Binance Futures
- строит свечные графики
- даёт фильтры по объёму, изменению, сделкам, волатильности и корреляции
- хранит вкладки и избранное в браузере
- обновляет последнюю свечу через WebSocket Binance

Главный файл фронтенда:

- `src/App.jsx`

## Что делает сервер

- загружает свечи Binance
- хранит историю в SQLite
- отдаёт данные фронтенду по HTTP
- догружает новые свечи и пытается заполнять пропуски

Главный файл сервера:

- `server.js`

## Основные API сервера

- `GET /ping`
- `GET /download-status`
- `GET /symbols`
- `GET /tickers`
- `GET /stats?tf=5m`
- `GET /klines?symbol=BTCUSDT&interval=5m`
- `GET /db-stats`
- `POST /redownload?symbol=...`
- `POST /fix-gaps`

## Важное замечание

Старые файлы в папке `Документация/`, где упоминаются 5 бирж, сохраняются только как исторические заметки. Для текущей версии проекта ориентируйся именно на этот файл, на `src/App.jsx` и на `D:\kopibar-server\server.js`.
