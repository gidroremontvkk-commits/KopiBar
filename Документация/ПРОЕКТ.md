# KopiBar — Документация проекта

## Что это такое
KopiBar — веб-приложение для мониторинга криптовалютных фьючерсных рынков.
Показывает графики свечей по монетам с 5 бирж, с фильтрацией по объёму, волатильности, корреляции с BTC и другим параметрам.

---

## Архитектура

```
Браузер (React) → Прокси-сервер (Node.js на VPS) → Биржи (Binance, Bybit, OKX, Gate.io, Bitget)
```

Прокси-сервер нужен чтобы не получать баны (418/429) от бирж при большом количестве запросов.
Сервер кэширует данные в памяти (TTL 60 сек), раздаёт их React-приложению.

## Фронтенд (React)

- **Файл:** `src/App.js`
- **Стили:** `src/App.css` (стили вкладок бирж добавлены в конец)
- **Библиотека графиков:** `lightweight-charts`
- **Адрес сервера в коде:** `const SERVER = 'http://77.239.105.144:3001'`

### Что умеет приложение:
- Выбор биржи через вкладки в шапке (Binance / Bybit / OKX / Gate.io / Bitget)
- Вкладки фильтров (можно создавать несколько наборов фильтров)
- Фильтрация по: объёму, изменению %, сделкам, NATR, волатильности, корреляции с BTC
- Сортировка по объёму / изменению / сделкам
- Виртуальный рендер (графики рисуются только когда видны на экране)
- Полноэкранный режим графика (кнопка ⛶)
- WebSocket обновление последней свечи в реальном времени (только Binance)
- Переключение таймфрейма глобально и для каждого графика отдельно
- Сохранение настроек вкладок в localStorage

## Очередь запросов (throttledFetch)
В App.js реализована очередь — не более 3 одновременных запросов, пауза 200мс.
Это защита от банов на случай прямых запросов (сейчас все запросы идут через сервер).

## Известные особенности и ограничения

- **OKX:** лимит 300 свечей за запрос (у остальных 1000), поэтому делается 13 запросов вместо 4
- **Gate.io / Bitget:** поле `count` (количество сделок) не возвращается API — всегда 0
- **WebSocket реалтайм:** реализован только для Binance. Остальные биржи обновляются через кэш (раз в минуту при новом запросе)
- **BTC корреляция:** определяется по символу начинающемуся на "BTC", подстраивается под формат каждой биржи
- **Кэш:** хранится в памяти RAM, сбрасывается при перезапуске сервера
